# Axioms Addressed

This repo is tied to immutable StoneyTECH axioms:

- [Axiom #2 — Push work down toward determinism](https://stoneytech.net/axioms#push-toward-determinism)
  - Review logic lands in deterministic rules, standards mappings, and reproducible CLI/MCP outputs rather than soft review mood.

- [Axiom #4 — GVR before pasting](https://stoneytech.net/axioms#gvr-before-pasting)
  - The scanner exists to review diffs, merges, and repositories before promotion or publication.

- [Axiom #5 — Never trust 'running' without sentinels](https://stoneytech.net/axioms#never-trust-running-without-sentinels)
  - Merge audits, audit history, trend views, and standards checks create sentinels around compliance posture.

- [Axiom #11 — Cite or be silent](https://stoneytech.net/axioms#cite-or-be-silent)
  - Findings carry named standards and controls so output can be inspected, challenged, and traced.

- [Axiom #13 — Ship with the failure mode named](https://stoneytech.net/axioms#ship-with-the-failure-mode-named)
  - The scanner names concrete failure classes: hardcoded secrets, prompt injection, unsafe token handling, git-ref abuse, and policy drift.

- [Axiom #16 — Don't comment without building. Don't curate without proving.](https://stoneytech.net/axioms#curate-and-prove)
  - The standards catalog is backed by runnable checks, tests, and an audit schema rather than a static list of good intentions.

- [Axiom #17 — Threat-model the surface](https://stoneytech.net/axioms#threat-model-the-surface)
  - Inputs, paths, refs, tokens, and MCP tool effects are treated as adversarial surfaces.

- [Axiom #21 — Scope before sharing](https://stoneytech.net/axioms#scope-before-sharing)
  - Public code and templates are shareable. Populated ledgers, local runtime state, repository clones, and credentials are not.
